## 服务端防火墙

在开始一切之前，需要确保服务端的防火墙允许正常端口通信，同时屏蔽其他端口以保证安全性。这是内网穿透服务的基础，必须做好。

| 授权策略 | 优先级 | 协议 | 端口 | 说明 |
| -------- | ------ | ---- | ---- | ---- |
| 允许     | 高     | TCP  | 22  | SSH 登录 |
| 允许     | 高     | TCP  | 443 | HTTPS 服务 |
| 允许     | 高     | TCP  | 7000 | frp 服务端 |
| 允许    | 高     | ICMP | | 允许 Ping |
| 拒绝     | 低     | ALL  | ALL  | 拒绝其他所有流量 |

## 服务端安装

前往 [Releases 页面](https://github.com/fatedier/frp/releases) 查看最新版本，下载对应的 Linux 版本。

```bash
wget https://github.com/fatedier/frp/releases/download/v0.66.0/frp_0.66.0_linux_amd64.tar.gz -O /tmp/frp.tar.gz
tar -xzvf /tmp/frp.tar.gz
mv /tmp/frp_0.66.0_linux_amd64/frps /usr/local/bin/frps
chmod +x /usr/local/bin/frps
```

新建配置文件 `/data/frp/frps.toml`，内容如下：

```toml
bindPort = 7000
auth.token = "<Var name="frp_token" />"
```

> 记得使用 [随机密码生成工具](https://tool.chinaz.com/tools/randompwd) 生成 `auth.token`，以保证安全性

新建 frp systemd 服务：

```bash
cat <<EOF >/etc/systemd/system/frps.service
[Unit]
Description = frp server
After = network.target syslog.target
Wants = network.target

[Service]
Type = simple
Restart = on-failure
RestartSec = 5s
ExecStart = /usr/local/bin/frps -c /data/frp/frps.toml

[Install]
WantedBy = multi-user.target
EOF
```

加载并启动 frp 服务：

```bash
systemctl daemon-reload
systemctl enable frps
systemctl start frps
systemctl status frps
```

## 客户端安装

与 [服务端安装](#服务端安装) 选择对应的 Linux 版本下载：

```bash
wget https://github.com/fatedier/frp/releases/download/v0.66.0/frp_0.66.0_linux_amd64.tar.gz -O /tmp/frp.tar.gz
tar -xzvf /tmp/frp.tar.gz
mv /tmp/frp_0.66.0_linux_amd64/frpc /usr/local/bin/frpc
chmod +x /usr/local/bin/frpc
```

> 注意，客户端用的是 `frpc`

新建配置文件 `/data/frp/frpc.toml`，内容如下：

```toml
serverAddr = "<Var name="server_addr" />"
serverPort = 7000
auth.token = "<Var name="frp_token" />"

[[proxies]]
name = "vscode-tcp"
type = "tcp"
localIP = "127.0.0.1"
localPort = 8886
remotePort = 8886

[[proxies]]
name = "rdp-tcp"
type = "tcp"
localIP = "127.0.0.1"
localPort = 11000
remotePort = 11000
```

> 记得将 `serverAddr` 替换为你的服务器地址，`auth.token` 替换为服务端配置的 token

这里我们绑定了两个端口：

| 代理名称    | 说明                     |
| ----------- | ------------------------ |
| vscode-tcp  | 将服务器的 8886 端口映射到本地的 8886 端口，用于 VS Code 远程开发 |
| rdp-tcp     | 将服务器的 11000 端口映射到本地的 11000 端口，用于远程桌面连接 |

新建 frp systemd 服务：

```bash
cat <<EOF >/etc/systemd/system/frpc.service
[Unit]
Description = frp client
After = network.target syslog.target
Wants = network.target

[Service]
Type = simple
Restart = on-failure
RestartSec = 5s
ExecStart = /usr/local/bin/frpc -c /data/frp/frpc.toml

[Install]
WantedBy = multi-user.target
EOF
```

加载并启动 frp 服务：

```bash
systemctl daemon-reload
systemctl enable frpc
systemctl start frpc
```

稍等一会儿，运行以下命令查看状态：

```bash
systemctl status frpc
```

## Nginx 配置

参考 <PostLink href="/linux/nginx" /> 部署好服务端的 Nginx，并配置好 SSL 证书后，添加以下反向代理配置：

然后我们就可以跟客户端一样配置 nginx 反向代理了，比如 VS Code server：

```nginx
server {
    listen 443 ssl;
    server_name <Var name="code_domain" />;

    include /etc/nginx/snippets/<Var name="domain" />-ssl.conf;
    include /etc/nginx/snippets/authelia.conf;

    location / {
        proxy_pass http://localhost:8886;
        
        include /etc/nginx/snippets/proxy-headers.conf;
        include /etc/nginx/snippets/websocket.conf;
    }
}
```

> 注意这里的 `server_name` 使用了 `<Var name="code_domain" />`，你需要提前配置好对应的 DNS 解析
